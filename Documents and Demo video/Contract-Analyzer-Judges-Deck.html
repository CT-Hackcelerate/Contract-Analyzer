Contract Analyzer · Overview

Read the contract. Decide with confidence.

Use case: first-pass risk review of customer / healthcare services & SDLC contracts (SOW, MSA, GSA, Order Forms).


The problem

Manual contract & SLA review takes hours, varies reviewer-to-reviewer, and buries penalties, uptime SLAs and commercial terms across dozens of pages. Leadership needs a fast, consistent proceed / renegotiate call.

The solution

An AI agent, built with Claude Code: upload a PDF/DOCX, it extracts and analyzes the contract under a strict grounded prompt, and produces a management-ready 8-page dashboard in ~10–15 minutes.

Business impact

~90% less first-pass effort; hours → minutes; every finding evidence-backed and reproducible; penalty & SLA exposure surfaced before signature. Applies across payers, providers, MedTech & SDLC.

Grounded

The model uses only the uploaded contract; missing values render "Not stated," never invented. Every risk quotes the clause + page + confidence.

Verified in code

Risk counts, status, money, page numbers, EM↔pricing and SLA benchmarks are recomputed in code — not trusted from the model.

Deterministic

Temperature 0 + a content-addressed cache: the same contract always yields the same report; duplicates never re-spend tokens.

Well made & safe

Guardrails & validation controls — and why

ControlWhy it's there
Input validationPDF/DOCX only, 25 MB cap, safe file name — blocks malformed, oversized or path-traversing uploads.
Prompt-injection / jailbreak screeningContract wrapped as untrusted data in markers + hardened system prompt — hidden "ignore instructions" text can't hijack the agent; hits are logged.
PII / secret maskingEmails, phones, PAN, Aadhaar, SSN, MRN, API keys, passwords redacted from every log — sensitive data never lands in a file.
PHI screeningEvery upload scanned for patient identifiers (MRN, DOB, SSN, ICD, NPI) — flagged & audited so no real patient data slips in.
Grounding + evidence"Not stated" for anything missing; a verbatim clause quote and confidence on every finding — prevents fabrication.
Deterministic re-computationCounts, status, money and page numbers computed in code — the numbers a manager reads are always correct.
EM ↔ pricing consistencyEngagement model and pricing forced to agree — an impossible pair (e.g. T&M + EM4 Fixed) can never display.
SLA benchmark (in code)Each SLA scored against a fixed playbook deterministically — verdicts can never be hallucinated.
Output schema validationModel JSON checked against the schema with one corrective retry — malformed output never reaches the UI.
Determinism (temp 0 + cache)Same contract → same result across runs; reproducible for judges and auditors.
Rate limitingPer-IP sliding window on /analyze — guards against abuse and runaway cost.
Access controlOnly the app pages & generated dashboards are served; prompt, source, cache, logs & history are not exposed over HTTP.
Audit trail + per-run logsEvery run appends a PII-masked audit line and writes a per-run log file — full traceability.

Reflection & roadmap

Lessons learnt & future enhancements

Lessons learnt

  • Ground everything and recompute critical facts in code — don't trust the model for numbers.
  • Determinism (temperature 0 + cache) is essential for trust and reproducibility.
  • Treat the uploaded contract as untrusted data — injection defense matters.
  • Keep the output template fixed; have the model return data only, not HTML.
  • Validate output + one corrective retry beats a single best-effort call.

Future enhancements

  • Multi-document analysis — MSA + SOW + annexures with cross-checks.
  • Configurable playbook per client / business unit with approval workflow.
  • Portfolio & batch mode + trend analytics.
  • Connectors & redline export — SharePoint and rewrites into Word.
  • SSO Login — Added security.
